## Overview

Valyu Research OS is a multi-agent research system that coordinates four specialised agents — each scoped to a different search domain — under a single team coordinator. A query sent to the team leader is automatically broken into sub-tasks, delegated to the appropriate agents, and synthesised into a structured intelligence report.

The system runs as a persistent FastAPI service on port `7777`, with all agent and team endpoints auto-generated by Agno's `AgentOS`. A separate Streamlit UI provides a mission-control interface with live agent activity panels.

## Agents

| ID | Label | Search Type | ValyuTools Config |
|---|---|---|---|
| `academic-search-agent` | ACADEMIC | Peer-reviewed journals, preprints, citations | `enable_academic_search=True` |
| `web-search-agent` | WEB | Live web, news, official docs | `enable_web_search=True` |
| `paper-deepdive-agent` | DEEP DIVE | Methodology, results, sections from specific papers | `enable_paper_search=True` |
| `domain-scoped-agent` | TRUSTED | nature.com · pubmed · arxiv · science.org · thelancet.com | `search_domains=[...]` |

Each agent is isolated — it only has access to its own search type. The team coordinator decides which agents to activate per query.

## Quickstart

### 1. Clone and install

```bash
git clone https://github.com/your-username/Valyu_research_os.git
cd Valyu_research_os

python -m venv valyu_env
valyu_env\Scripts\activate        # Windows
# source valyu_env/bin/activate   # macOS/Linux

pip install -r requirements.txt
```

### 2. Configure environment

```bash
cp .env.example .env
```

Edit `.env`:

```bash
ANTHROPIC_API_KEY=your_anthropic_key
VALYU_API_KEY=your_valyu_key
AGNO_API_KEY=your_agno_key          # optional — needed for os.agno.com control plane
```

Get your keys:
- **Anthropic** → [console.anthropic.com](https://console.anthropic.com)
- **Valyu** → [valyu.ai](https://valyu.ai)
- **Agno** → [app.agno.com](https://app.agno.com)

### 3. Start the backend

```bash
python app.py
```

Expected output:
```
INFO:     Uvicorn running on http://0.0.0.0:7777
INFO      Agno tracing successfully set up with database storage
INFO:     Application startup complete.
🚀 Valyu Research OS starting…
```

### 4. Start the Streamlit UI (separate terminal)

```bash
streamlit run valyu_ui.py
```

Opens at `http://localhost:8501`

## API Reference

The service runs at `http://localhost:7777`. All agent and team endpoints accept `application/x-www-form-urlencoded`.

### Custom routes

| Method | Path | Description |
|---|---|---|
| `GET` | `/` | API info and registered agents/teams |
| `GET` | `/health` | Health check |
| `GET` | `/status` | Service metadata |
| `GET` | `/agents/list` | List all registered agents |
| `GET` | `/teams/list` | List all registered teams |

### AgentOS auto-generated routes

| Method | Path | Description |
|---|---|---|
| `POST` | `/agents/{agent_id}/runs` | Run a single agent |
| `POST` | `/teams/{team_id}/runs` | Run the full research team |
| `POST` | `/agents/{agent_id}/runs/{run_id}/cancel` | Cancel an in-flight agent run |
| `POST` | `/teams/{team_id}/runs/{run_id}/cancel` | Cancel an in-flight team run |

Full interactive docs at `http://localhost:7777/docs`

## Usage Examples

### Health check
```bash
curl http://localhost:7777/health
```

### Run the full research team
```bash
curl -X POST "http://localhost:7777/teams/valyu-research-team/runs" \
  -H "Content-Type: application/x-www-form-urlencoded" \
  --data-urlencode "message=What are the latest advances in GLP-1 receptor agonists for treating obesity?" \
  --data-urlencode "stream=false" \
  --data-urlencode "session_id=session-001"
```

### Run with streaming (tokens arrive as agents work)
```bash
curl -X POST "http://localhost:7777/teams/valyu-research-team/runs" \
  -H "Content-Type: application/x-www-form-urlencoded" \
  --data-urlencode "message=Summarise recent CRISPR breakthroughs in rare diseases" \
  --data-urlencode "stream=true" \
  --data-urlencode "session_id=session-001" \
  --no-buffer
```

### Follow-up in the same session (tests memory)
```bash
curl -X POST "http://localhost:7777/teams/valyu-research-team/runs" \
  -H "Content-Type: application/x-www-form-urlencoded" \
  --data-urlencode "message=Summarise the top 3 key takeaways from what you just found" \
  --data-urlencode "stream=false" \
  --data-urlencode "session_id=session-001"
```

### Run a single agent directly
```bash
curl -X POST "http://localhost:7777/agents/academic-search-agent/runs" \
  -H "Content-Type: application/x-www-form-urlencoded" \
  --data-urlencode "message=Find recent papers on transformer architectures for protein folding" \
  --data-urlencode "stream=false"
```

## Frontend Integration

The service is a standard ASGI application — any frontend can consume it.

### JavaScript (fetch + streaming)
```javascript
async function runResearch(query, sessionId, onToken) {
  const res = await fetch("http://localhost:7777/teams/valyu-research-team/runs", {
    method: "POST",
    headers: { "Content-Type": "application/x-www-form-urlencoded" },
    body: new URLSearchParams({
      message: query,
      stream: "true",
      session_id: sessionId,
    }),
  });

  const reader = res.body.getReader();
  const decoder = new TextDecoder();

  while (true) {
    const { done, value } = await reader.read();
    if (done) break;
    onToken(decoder.decode(value));
  }
}
```

## Model Configuration

The team coordinator requires a model that supports **tool calling** — this is how it delegates tasks to member agents internally. Individual agents only need to call `ValyuTools`, which has lighter requirements.

| Role | Recommended model | Notes |
|---|---|---|
| Team coordinator | `claude-sonnet-4-6` | Best delegation and synthesis quality |
| Team coordinator | `meta-llama/Meta-Llama-3.1-70B-Instruct` | Free on HuggingFace, 128k context |
| Team coordinator | `Qwen/Qwen2.5-72B-Instruct` | Strong open-source tool calling |
| Member agents | Any HuggingFace model | `meta-llama/Meta-Llama-3-8B-Instruct` works fine |

> **Important:** `meta-llama/Meta-Llama-3-8B-Instruct` (without `.1`) does **not** support tool calling and will fail as a team coordinator. Use it for member agents only.

## Session Memory

All agents and the team share a `SqliteDb` instance. Sessions persist across server restarts.

```python
db = SqliteDb(db_file="tmp/research_os.db")
```

Pass the same `session_id` across requests to maintain conversation context:

```
Request 1: session_id=abc → agent stores run in db
Request 2: session_id=abc → agent loads prior run from db, answers in context
```

The `tmp/research_os.db` file is created automatically on first run and is excluded from version control via `.gitignore`.

## AgentOS Control Plane

Connect `os.agno.com` to your running service for a live session dashboard, run inspector, and tracing.

1. Ensure no `OS_SECURITY_KEY` is set in `.env`
2. Start the server with `tunnel=True` for a public URL:

```python
# app.py
agent_os.serve(app="app:app", host="0.0.0.0", port=7777, reload=True, tunnel=True)
```

Or use [ngrok](https://ngrok.com):

```bash
ngrok http 7777
```

Then go to `os.agno.com` → Connect → paste the public URL.
